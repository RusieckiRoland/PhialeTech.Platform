# ADR: WebAssets NuGet as Single Source of JS Assets

* **Status:** Accepted
* **Date:** 2026-03-07
* **Owners:** GIS Platform Team
* **Related:** PhialeGis.WebAssets, Avalonia/WPF/WinUI3/UWP UI libraries

---

## Context

JS assets for the DSL editor (Monaco) were duplicated across platform projects:

* some projects copied from `WinUI3/.../Assets/Monaco`
* some projects copied from `PhialeGis.MonacoAsets/.../Assets/Monaco`

This caused drift risk, harder maintenance, and inconsistent runtime packaging.

---

## Decision

1. We standardize on **one source of truth**: `PhialeGis.WebAssets` NuGet package.
2. Platform UI libraries and sandboxes consume JS assets via `PackageReference` to `PhialeGis.WebAssets`.
3. Repository-level `NuGet.config` defines a **local feed** at `.nuget/local-feed`.
4. `PhialeGis.WebAssets` packs to `.nuget/local-feed` (`PackageOutputPath`).
5. Asset layout convention is mandatory:
   `Assets/<LibraryName>/...` (for example `Assets/Monaco/...`, future `Assets/Leaflet/...`).
6. Host/adapter scripts are kept next to the relevant library assets to avoid cross-library coupling.

---

## Rationale

* Eliminates platform-specific asset duplication.
* Gives deterministic versioning and rollback via NuGet versions.
* Works offline (no CDN dependency).
* Simplifies onboarding: repo already contains source config for restore.
* Scales to additional JS libraries without changing distribution model.

---

## Consequences

### Positive

* All platforms load assets from the same packaged source.
* Asset updates happen in one place.
* Better CI/CD reproducibility and lower regression risk.

### Trade-offs

* Requires package version management.
* Local feed must contain the built `.nupkg`.
* Package naming may no longer reflect only Monaco, therefore generalized to `WebAssets`.

---

## Implementation Notes

* Package project renamed from `MonacoAssets` to `WebAssets`.
* `PackageId` renamed to `PhialeGis.WebAssets`.
* Direct file-link includes to Monaco assets were removed from consumer projects.
* Consumers now use `PackageReference Include="PhialeGis.WebAssets"`.
* Local feed configured in `NuGet.config`.

---

## Validation

For each target platform:

1. restore succeeds using repo `NuGet.config`
2. output contains `Assets/Monaco/...`
3. DSL editor loads Monaco host and worker files from packaged assets

---

## Future Extension Rule

When adding new JS library assets:

1. add under `Assets/<LibraryName>/...` in `PhialeGis.WebAssets`
2. keep host/bridge scripts near that library
3. bump package version
4. update consumers through NuGet version change only
