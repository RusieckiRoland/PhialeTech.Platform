# ADR: UWP Moved to Legacy; Active UI Development Focuses on WinUI 3, Avalonia, and WPF

* **Status:** Accepted
* **Date:** 2026-03-10
* **Owners:** GIS Platform Team
* **Related:** `legacy/Uwp`, WinUI 3, Avalonia, WPF, platform UI libraries

---

## Context

The repository currently contains multiple desktop UI technology paths, including:

* UWP
* WinUI 3
* Avalonia
* WPF

UWP is no longer aligned with the active product direction for new UI work. At the same time, the team needs a clear statement about which desktop UI platforms are still under active development and where engineering effort should be concentrated.

We also need an explicit platform priority rule so that:

* new component work is not split across obsolete targets,
* platform investment is focused on currently viable UI stacks,
* expectations around Avalonia are stated clearly, especially regarding touch support,
* older UWP code can remain in the repository without implying ongoing feature development.

---

## Decision

### 1. UWP is moved to Legacy

UWP is formally classified as a **legacy platform** in this repository.

It is no longer part of the active development path for new components or new feature work.

The UWP codebase may remain in the repository for:

* reference
* historical compatibility
* controlled maintenance if strictly necessary

It must not be treated as an active target for ongoing product evolution.

### 2. No new feature development will target UWP

Effective immediately:

* no new reusable components will be built for UWP,
* no platform expansion work will prioritize UWP,
* no new product-path work will use UWP as a primary target.

Bug fixes or maintenance actions for UWP should be exceptional and justified by a concrete business need.

### 3. Active UI platform focus is WinUI 3, Avalonia, and WPF

Current active UI development focuses on:

* **WinUI 3**
* **Avalonia**
* **WPF**

These are the platforms that should receive architectural attention, host implementations, testing effort, and feature delivery for current and future UI components.

### 4. Avalonia is supported with a cautious platform expectation

Avalonia remains an active target platform.

However, platform investment in Avalonia should be made with an explicit awareness that:

* the team is awaiting **Avalonia 12** with particular interest,
* full touch support is expected to become materially stronger there,
* touch-heavy UX decisions for Avalonia should therefore be made cautiously until that platform state is confirmed in practice.

This is not a suspension of Avalonia work. It is an explicit statement that touch-oriented investment on Avalonia should be approached carefully until the Avalonia 12 capability level is validated.

---

## Rationale

* UWP no longer represents the forward-looking desktop UI direction of the product.
* Keeping UWP in the active target set would dilute engineering effort.
* WinUI 3, Avalonia, and WPF are the platforms with current strategic value.
* WPF remains important for existing desktop delivery.
* WinUI 3 represents the main Microsoft-native forward path.
* Avalonia remains strategically relevant for cross-platform reach, but touch readiness must be evaluated pragmatically.

---

## Consequences

### Positive

* Platform priorities become explicit.
* New component development avoids wasting effort on UWP.
* Repository structure can reflect the real status of platforms.
* Planning for WPF, WinUI 3, and Avalonia becomes more coherent.

### Trade-offs


* Any future return to UWP would require an explicit reversal of this decision.
* Avalonia touch-oriented UX may progress more cautiously than other platform work until Avalonia 12 is validated.

---

## Repository Rule

UWP should be treated as a legacy path and placed under:

```text
legacy/Uwp/
```

Active platform work should be organized under current product and platform paths for:

```text
Platforms/Wpf/
Platforms/WinUI3/
Platforms/Avalonia/
```

---

## Enforcement Rule

Any proposal for new component work must assume:

1. UWP is not an active target.
2. WPF, WinUI 3, and Avalonia are the active desktop UI targets.
3. Avalonia touch-heavy UX decisions require explicit caution until Avalonia 12 capability is validated.

If a proposal includes UWP as a default active target, it conflicts with this decision.
