# ADR-001: Prioritizing Code Readability Over Advanced Capabilities

## Status
Applied

## Context
While designing the data structures for my SkiaSharp library, I deliberated over the use of field offsets (`FieldOffset`) to achieve more flexibility. This technique could afford me greater control over the memory layout but might also complicate maintenance and future development.

## Decision
I have decided to adopt a simpler, sequential memory layout. This decision is based on my belief that code readability and maintainability are more important than the potential benefits that come with having finer control over memory layout, especially considering the current and anticipated usage of the library.

## Consequences
- **Positive**:
  - The code I write is more understandable for myself and for anyone who might join the project in the future.
  - I am able to implement changes and update the code more swiftly.
  - There is a reduced risk of unintended memory management errors.
- **Negative**:
  - I have to accept that some memory-level optimizations will not be possible without restructuring the data structures.

## Notes
- I will continue to monitor the application's performance, and if I find the need for more control over memory layout, I may revisit this decision.
