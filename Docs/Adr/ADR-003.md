# ADR-003: Multi-Project Structure for Cross-Platform UI Compatibility

## Status
Applied (2024-01-22), updated to current repository state (2026-02-16)

## Context
PhialeGis requires a structure that keeps core GIS and DSL logic platform-agnostic while supporting multiple UI technologies.

The current repository supports:
- WPF
- UWP
- WinUI 3
- Avalonia

The DSL editor pipeline must stay consistent across all hosts.

## Decision
The solution is organized into clear layers:

1. **Core shared logic**
- `PhialeGis.Library.Core` (`netstandard2.0`) for UI-agnostic GIS runtime and model logic.

2. **Shared rendering layer**
- `PhialeGis.Library.Renderer.Skia` (`netstandard2.0`) for Skia-based rendering logic.

3. **Shared DSL/editor orchestration**
- `PhialeGis.Library.Dsl` for grammar, parsing, diagnostics, and visitor mapping.
- `Features/PhialeGis.Library.DslEditor` for editor-runtime integration services.

4. **Platform UI adapters**
- `Wpf/PhialeGis.Library.WpfUi` (`net6.0-windows`, WebView2 host).
- `Uwp/PhialeGis.Library.UwpUi` (UAP project, WebView2 host).
- `WinUI3/PhialeGis.Library.WinUi` (`net8.0-windows10.0.19041.0`, WebView2 host).
- `Avalonia/PhialeGis.Library.AvaloniaUi` (`net8.0`, `CefGlue.Avalonia` host).

5. **Shared Monaco assets**
- Monaco assets are maintained in `PhialeGis.WebAssets/Assets/Monaco` and consumed by UI hosts.

## Consequences
- **Positive**:
  - Strong separation between domain logic and host UI concerns.
  - Easier parity of DSL editor behavior across platforms.
  - Clear upgrade path for adding new hosts without rewriting core logic.
- **Negative**:
  - More projects to keep synchronized.
  - Bridge and hosting differences (WebView2 vs CEF) require targeted maintenance.

## Alternatives Considered
- A single UI project with conditional compilation (rejected due to complexity and poor readability).
- Merging rendering into Core (rejected to preserve separation of concerns).

## Notes
- This update supersedes older naming used in the original ADR text (`PhialeGis.Library.Drawing`, `WpfUI`, `UwpUI` as .NET Core class libraries).
- The principle remains unchanged: shared core + platform-specific UI adapters.
